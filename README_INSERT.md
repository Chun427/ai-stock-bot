## 交易定義（Trading Definition）

系統的預測目標與績效統計採用**同一套規則**（Performance Metric Alignment）：

| 項目 | 定義 |
|---|---|
| **Entry（基準價）** | 推播當日採用「前一交易日收盤價」為基準價 |
| **Holding（持有期）** | 最多 **5 個交易日** |
| **Take Profit（停利）** | 任一日最高價 ≥ 買價 **+3%** → 出場，報酬記 **+3%** |
| **Stop Loss（停損）** | 任一日最低價 ≤ 買價 **−1.5%** → 出場，報酬記 **−1.5%**（優先於停利） |
| **Time Exit（時間出場）** | 5 日內皆未觸發 → 第 5 交易日收盤出場，報酬為當日收盤相對買價 |
| **Success（命中）** | 5 日內先觸 +3% 且未先觸 −1.5% → `label = 1` |

### Performance Metrics

- **命中率**：達成 +3% 的預測數 / 總預測數
- **平均報酬**：依上述出場規則計算之每筆平均報酬
- **最大虧損**：單筆最差報酬（受停損保護）
- **正報酬比例**：報酬 > 0 的比例

> ⚠️ **已知限制**
> 1. 基準價為前一交易日收盤價，實際須於隔日開盤後買入，故存在**跳空缺口**，
>    回測/驗證績效可能偏樂觀（待後續版本處理）。
> 2. 停利/停損假設可於觸價成交，未計滑價。
> 3. 命中率與報酬皆需足夠樣本（建議 ≥ 50–100 筆）方具統計意義。
>    在此之前，**不應據以判斷模型是否具 Alpha**。
